# 代码审查检查清单

## 逻辑正确性

- [ ] 业务逻辑是否符合需求描述
- [ ] 边界条件是否已处理（空值、零值、最大值、空数组）
- [ ] 分支覆盖是否完整，是否存在遗漏的 else 分支
- [ ] 循环终止条件是否正确，是否存在死循环风险
- [ ] 并发场景下数据一致性是否保障

## 安全性

- [ ] 用户输入是否经过校验和转义
- [ ] 是否存在 SQL 注入、XSS、CSRF 风险
- [ ] 敏感信息是否避免硬编码和日志输出
- [ ] 权限校验是否覆盖所有入口
- [ ] 第三方依赖是否存在已知漏洞

## 性能

- [ ] 是否存在 N+1 查询
- [ ] 大列表是否使用分页或虚拟滚动
- [ ] 是否有不必要的重复计算或重复请求
- [ ] 缓存策略是否合理
- [ ] 内存使用是否有泄漏风险

## 可维护性

- [ ] 命名是否清晰表达意图
- [ ] 函数职责是否单一，长度是否合理
- [ ] 是否存在重复代码可提取为公共方法
- [ ] 复杂逻辑是否有注释说明
- [ ] 类型定义是否准确，是否避免了 any

## 错误处理

- [ ] 异步操作是否正确捕获异常
- [ ] 错误信息是否对调试有帮助
- [ ] 是否存在空 catch 块
- [ ] 用户可见错误信息是否友好
- [ ] 错误恢复路径是否合理

## 测试

- [ ] 关键路径是否有单元测试
- [ ] 测试用例是否覆盖正常、异常、边界场景
- [ ] Mock 数据是否合理，是否过度 Mock
- [ ] 集成测试是否覆盖核心流程
