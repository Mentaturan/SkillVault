# 重构检查清单

## 重构前

- [ ] 明确重构目标：性能优化、可读性提升、架构调整、消除重复
- [ ] 确认现有测试覆盖是否足够，不足则先补充测试
- [ ] 确认重构范围，避免在重构中混入功能变更
- [ ] 创建功能分支，确保可随时回退
- [ ] 记录当前代码行为快照，作为回归对照

## 重构中

- [ ] 小步提交，每步保持可编译可运行
- [ ] 每步提交后运行测试确认无回归
- [ ] 重命名时使用 IDE 的 Rename 功能，确保引用全部更新
- [ ] 提取方法时保持原方法签名不变，内部委托到新方法
- [ ] 移动代码时先复制后删除，避免丢失
- [ ] 删除死代码前确认无反射调用、无配置引用

## 常见重构模式

### 提取方法

- 识别长方法中的逻辑段落
- 提取为独立方法，方法名描述意图而非实现
- 参数不超过 3 个，超过则考虑引入参数对象

### 消除重复

- 识别三处以上相同逻辑
- 提取公共方法，保留差异通过参数控制
- 避免过度抽象，两处重复可暂不提取

### 简化条件

- 嵌套 if 转为卫语句（early return）
- 复杂条件提取为语义化的布尔变量或方法
- 策略模式替代过长的 switch-case

### 引入类型

- 字面量联合类型超过 3 个值转为 enum
- 散落的对象结构定义为 interface
- any 类型逐步收窄为具体类型

## 重构后

- [ ] 全量测试通过
- [ ] 手动验证核心流程
- [ ] 检查性能指标无退化
- [ ] 更新相关注释和文档
- [ ] Code Review 重点审查重构范围是否超出预期
- [ ] 确认无功能变更混入
